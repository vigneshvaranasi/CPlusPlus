Classes and Objects

Class

A class is a user-defined data type that serves as a blueprint for creating objects. It encapsulates data (attributes) and functions (methods) that operate on that data. Classes provide a way to structure and organize code in an object-oriented paradigm, supporting principles such as encapsulation, inheritance, and polymorphism.

Class Declaration:

  • A class is declared using the class keyword, followed by the class name and a block that contains the class members.

Data Members (Attributes):

  • Variables within a class are called data members or attributes.
  • They define the properties or characteristics of objects created from the class.

Member Functions (Methods):

  • Functions within a class are called member functions or methods.
  • They define the behavior or actions that objects of the class can perform.

Access Modifiers:

  • public, private, and protected are access modifiers that control the visibility of class members.
  • public: Accessible from outside the class.
  • private: Accessible only within the class.
  • protected: Similar to private but accessible in derived classes.

Object

Object is an instance of a class. Objects encapsulate data (attributes or properties) and the functions (methods) that operate on that data. They provide a way to represent and interact with real-world entities in a program using the principles of object-oriented programming (OOP)

Class Definition:

  • Objects are created based on a class, which serves as a blueprint or template defining the structure and behavior of objects.

Object Creation:

  • Objects are instances of a class, created by specifying the class name followed by an object name.

Accessing Members:

  • Members (data and functions) of an object are accessed using the dot (.) operator.

Data Members (Attributes):

  • Data members represent the properties or characteristics of objects. They define the state of the object.

Member Functions (Methods):

  • Member functions define the behavior or actions that objects of the class can perform.

Constructor:

  • A special member function called when an object is created. It initializes the object's properties.

Destructor:

  • A special member function called when an object goes out of scope. It cleans up resources and performs necessary tasks.

Initialization and Assignment:

  • Objects can be initialized during creation or assigned values later.

Object Copying:

  • Objects can be copied using the copy constructor or the assignment operator.

Access Specifiers and their scope

Public:

  • Keyword: public
  • Scope: Public members are accessible from anywhere in the program, both inside and outside the class.
  • Purpose: Public members are meant to be part of the class's interface, providing a way for external code to interact with the class.

Private:

  • Keyword: private
  • Scope: Private members are only accessible within the class. They are not visible or accessible outside the class.
  • Purpose: Private members encapsulate the internal details of the class, hiding them from external code.

Protected:

  • Keyword: protected
  • Scope: Protected members are similar to private members but have limited accessibility in derived classes. They are visible to derived classes but not to the external code.
  • Purpose: Protected members provide a level of access that is between public and private, often used in the context of inheritance.

Member Function

A member function is a function that is associated with a specific class or object. Member functions define the behavior or actions that can be performed on objects of a class. They are an essential part of object-oriented programming (OOP) and contribute to the encapsulation of data and functionality within a class

Declaration and Definition:

  • Member functions are declared within the class definition and defined outside the class. The declaration provides the function's signature, while the definition contains the actual implementation.

Access Specifiers:

  • Member functions can have access specifiers (public, private, or protected) that control their visibility and accessibility.

Calling Member Functions:

  • Member functions are called using the dot (.) operator on an object of the class.

Access to Data Members:

  • Member functions have access to the data members (attributes) of the class. They can manipulate and operate on the state of the object.

Const Member Functions:

  • Member functions can be declared as const to indicate that they do not modify the state of the object. This is a good practice for functions that don't change the object's data.

Static Function

A static function is a member function of a class that belongs to the class itself rather than to instances (objects) of the class. It is associated with the class and not with any particular object. Static functions are declared using the static keyword in the class definition.

Declaration:

  • A static function is declared using the static keyword in the class definition.
  • It can be called on the class itself, not requiring an instance of the class.

Use Cases:

  • Utility Functions: Static functions are often used for utility functions that don't rely on the state of individual objects.
  • Counters: They can be used for maintaining shared counters or data that is common to all instances of the class.
  • Factory Methods: In some cases, static functions are used as factory methods to create instances of the class.

Accessing Static Members:

  • Static members (including static functions) are accessed using the class name and the scope resolution operator ::.

No Access to Non-Static Members:

  • Static functions cannot access non-static (instance) members directly. They operate on the class level and don't have access to a specific object's state

Use of Static Functions:

  • Shared Functionality: Static functions provide a way to define functionality that is shared among all instances of a class. They operate at the class level, not relying on the state of individual objects.
  • Memory Efficiency: Since static functions don't operate on object instances, they don't have access to instance-specific data. This can lead to more memory-efficient code.
  • Global Scope: Static functions can be called without creating an instance of the class, allowing for a global scope of functionality associated with the class itself.

Declaration of a Static Member:

To declare a data member of a class as static, the static keyword is used in the class definition. This indicates that the member belongs to the class itself, not to individual objects.

Friend Function

A friend function is a function that is not a member of a class but has access to its private and protected members. Friend functions are declared with the friend keyword within the class
definition, indicating that they are trusted functions that can access the class's private and protected members

Declaration:

  • Friend functions are declared within the class definition using the friend keyword.

Definition:

  • Friend functions are defined outside the class definition but still have access to the class's private and protected members.

Usage:

  • Friend functions are typically used when external functions need special access to the private or protected members of a class.

Access to Private and Protected Members:

  • Friend functions can access private and protected members of the class as if they were regular functions of the class.

Not a Member of the Class:

  • Friend functions are not members of the class but are declared and defined within the class. They do not have the class:: scope.

Global or in a Different Namespace:

  • Friend functions can be global functions or functions in a different namespace. They are not tied to the class namespace.

Use Cases:

  • Friend functions are useful when you want to provide external functions with privileged access to the internals of a class.
  • They are often used for operator overloading where non-member functions need access to private data.
