Introduction to C++

C++ is a powerful, versatile programming language developed as an extension of C. It supports both low-level memory manipulation and high-level abstractions, making it widely used for system software, games, and various applications. With features like classes and templates, C++ enables modular and efficient code development

What is OOPS?

OOPS stands for Object-Oriented Programming (OOP). It is a programming paradigm that uses the concept of "objects" to design and organize code. Objects are instances of classes, and a class is a blueprint or template that defines the attributes (data) and behaviors (methods/functions) common to all objects of a certain type.

Features of OOP

1. Objects:
Objects are instances of classes and represent real-world entities. They encapsulate data and behavior.

2. Classes:
Classes serve as blueprints or templates for creating objects, defining attributes and methods common to all instances.

3. Encapsulation:
Encapsulation bundles data and methods within a class, restricting direct access to the internal details and providing controlled interaction.

4. Data Abstraction:
Data abstraction involves hiding the implementation details of data and exposing only essential features, enhancing security and simplicity.

5. Inheritance:
Inheritance allows a class (subclass) to inherit properties and behaviors from another class (superclass), promoting code reuse and hierarchy.

6. Polymorphism:
Polymorphism enables objects of different types to be treated as objects of a common type, supporting flexibility and adaptability.

7. Dynamic Binding:
Dynamic binding allows the association between a method call and its implementation to be determined at runtime, promoting flexibility.

8. Message Passing:
Objects communicate by sending messages, facilitating interaction and collaboration between objects in a system.

9. Reusability:
Reusability is achieved through mechanisms like inheritance and polymorphism, reducing redundancy and promoting efficient code utilization.

10. Delegation:
Delegation involves one object assigning a task to another object, promoting a clear separation of responsibilities and modular design.

11. Genericity:
Genericity, or generics, allows the creation of classes or methods that can work with different types, promoting code flexibility and reuse.

Pre-Defined streams

  1. Standard Input:
    • This stream is used for reading input from the user or from another program.
    • It is represented by stdin.
  1. Standard Output:
    • This stream is used for writing output to the console or terminal.
    • It is represented by stdout.
  1. Standard Error:
    • This stream is used for writing error messages to the console or terminal.
    • It is represented by stderr.

Namespace

A namespace is a feature that allows you to organize code into separate logical units to avoid naming conflicts. It provides a way to group related code elements, such as variables, functions, and classes, under a specific name. This helps in preventing naming collisions, especially in larger projects where different parts of the code may have similar or identical names.

They are declared using the namespace keyword, followed by the namespace name and a block of code.

Passing Arguments

1. Passing by Value:
◦ Arguments are copied into the function's parameters.
◦ Changes to parameters do not affect the original values.

2. Passing by Reference:
◦ Function receives references to the original variables.
◦ Changes to parameters directly affect the original values.

3. Passing by Pointer:
◦ Function receives pointers to the memory addresses of variables.
◦ Allows direct manipulation of values using pointers.

4. Default Argument Values:
◦ Parameters can have default values, enabling omission during function calls.

5. Passing Arrays:
◦ Arrays are usually passed by reference or as pointers to their first elements.

6. Passing Objects:
◦ Objects can be passed by value or by reference, depending on the desired behavior.

7. Passing Constant Arguments:
◦ Parameters marked as
const prevent modification of the original values.

8. Passing Arguments in Constructors:
◦ Constructors of classes can receive arguments during object creation.

9. Passing Arguments in Function Templates:
◦ Function templates can accept arguments of various types, promoting generic programming.

Inline function

Declaration:

  • An inline function is typically declared in the same way as a regular function but with the inline keyword preceding the function declaration.

Advantages:

  • Reduced Function Call Overhead: Inlining eliminates the overhead of function calls, as the compiler inserts the function's code directly at the call site.
  • Performance Improvement: Can result in improved performance for small, frequently called functions.

Limitations:

  • Size Increase: Inlining may lead to larger code size due to duplicated code at each call site.
  • Compiler Decision: The compiler decides whether to honor the inline keyword; it may choose not to inline a function.

Guidelines:

  • Short Functions: Inline functions are most effective for short, simple functions.
  • Avoid in Large Functions: Avoid marking large functions as inline to prevent code bloat.
  • Use Profiling: Inline functions should be used judiciously and based on profiling results to ensure actual performance gains.

Function overloading

Function overloading in C++ allows multiple functions in the same scope to have the same name but with different parameter lists. The compiler determines the correct function to call based on the number and types of arguments provided. Function overloading enhances code readability, flexibility, and reusability.

Key Points:

  1. Same Name, Different Signatures:
    • Functions with the same name but different parameter lists, including a different number or type of parameters.
  1. Advantages:
    • Readability: Provides a clear and intuitive interface by using the same function name for related operations.
    • Flexibility: Allows functions to handle different types of data or perform slightly different tasks.
    • Default Arguments: Function overloading can work with default argument values.
  1. Conditions for Overloading:
    • Functions must have the same name.
    • Functions must have different parameter lists (number, type, or both).
    • Return type alone is not sufficient for overloading.
